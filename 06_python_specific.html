Python Fundamentals

Question 1: Explain the difference between list, tuple, and dictionary in Python.

Detailed Explanation

Lists:
  • Mutable, ordered collection
  • Can contain duplicate elements
  • Indexed by integers
  • Use [] syntax
  • Time complexity: O(1) for access, O(n) for insert/delete

List example

my_list = [1, 2, 3, 4, 5]
my_list.append(6)  # Mutable
my_list[0] = 10    # Can modify elements
print(my_list)      # [10, 2, 3, 4, 5, 6]

Tuples:
  • Immutable, ordered collection
  • Can contain duplicate elements
  • Indexed by integers
  • Use () syntax
  • Time complexity: O(1) for access
  • Faster than lists, use less memory

Tuple example

my_tuple = (1, 2, 3, 4, 5)

my_tuple[0] = 10  # Error: tuples are immutable


Use cases: coordinates, database records, function returns

point = (10, 20)

Dictionaries:
  • Mutable, unordered collection (Python 3.7+ maintains insertion order)
  • Key-value pairs
  • Keys must be hashable (immutable types)
  • Use {} syntax
  • Time complexity: O(1) average for access/insert/delete

Dictionary example

my_dict = {'name': 'John', 'age': 30, 'city': 'NYC'}
my_dict['email'] = 'john@example.com'  # Add new key-value
my_dict['age'] = 31  # Update value
print(my_dict)  # {'name': 'John', 'age': 31, 'city': 'NYC', 'email': 'john@example.com'}

When to Use Each:
  • List: When you need a mutable, ordered sequence
  • Tuple: When you need an immutable sequence (keys in dicts, function arguments)
  • Dictionary: When you need key-value mapping with fast lookups

Question 2: Explain Python's GIL (Global Interpreter Lock) and its implications.

Detailed Explanation

What is GIL:

The Global Interpreter Lock is a mutex that protects access to Python objects, preventing multiple native threads from executing Python bytecodes at once.

How it Works:
  • Only one thread can execute Python bytecode at a time
  • Threads must acquire GIL before executing Python code
  • GIL is released during I/O operations and C extension calls
Implications:
  • CPU-bound Tasks:
import threading
import time

def cpu_bound_task():
    total = 0
    for i in range(10000000):
        total += i
    return total


Multi-threading doesn't help for CPU-bound tasks

start = time.time()
threads = []
for _ in range(4):
    t = threading.Thread(target=cpu_bound_task)
    t.start()
    threads.append(t)

for t in threads:
    t.join()
print(f"Multi-threaded: {time.time() - start}")  # Similar to single-threaded
  • I/O-bound Tasks:
import threading
import requests

def io_bound_task(url):
    response = requests.get(url)
    return response.status_code


Multi-threading helps for I/O-bound tasks

urls = ['http://example.com'] * 10
threads = []
for url in urls:
    t = threading.Thread(target=io_bound_task, args=(url,))
    t.start()
    threads.append(t)

GIL is released during I/O, so threads can run concurrently
Solutions:
  • Multiprocessing (for CPU-bound):
from multiprocessing import Pool

def cpu_bound_task(n):
    total = 0
    for i in range(n):
        total += i
    return total

if __name__ == '__main__':
    with Pool(4) as pool:
        results = pool.map(cpu_bound_task, [10000000] * 4)
  • Async/Await (for I/O-bound):
import asyncio
import aiohttp

async def io_bound_task(session, url):
    async with session.get(url) as response:
        return await response.status

async def main():
    async with aiohttp.ClientSession() as session:
        urls = ['http://example.com'] * 10
        tasks = [io_bound_task(session, url) for url in urls]
        results = await asyncio.gather(*tasks)
    return results

asyncio.run(main())

  • Shows awareness of Python's limitations and solutions

Question 3: Explain Python's memory management and garbage collection.

Detailed Explanation

Memory Management:
  • Reference Counting:
- Each object has a reference count

- Incremented when referenced, decremented when dereferenced

- Object deleted when count reaches 0

import sys

a = [1, 2, 3]
print(sys.getrefcount(a))  # Reference count (includes temporary ref in getrefcount)

b = a  # Reference count increases
print(sys.getrefcount(a))  # Increased

del b  # Reference count decreases
print(sys.getrefcount(a))  # Decreased

  • Garbage Collection:
- Handles circular references

- Uses generational garbage collection

- Three generations (0, 1, 2)

import gc


Circular reference example

class Node:
    def __init__(self, value):
        self.value = value
        self.ref = None

a = Node(1)
b = Node(2)
a.ref = b
b.ref = a  # Circular reference


Reference counting can't handle this


Garbage collector will eventually clean it up

del a, b
gc.collect()  # Force garbage collection

Memory Optimization: 

Use __slots__ to reduce memory usage

class Point:
    __slots__ = ['x', 'y']  # Prevents __dict__ creation
    
    def __init__(self, x, y):
        self.x = x
        self.y = y


Without __slots__: ~56 bytes per instance


With __slots__: ~40 bytes per instance

Question 4: Implement a custom data structure (e.g., LRU Cache).

Problem Statement

Implement an LRU (Least Recently Used) Cache.

Detailed Explanation

Requirements

  • get(key): Return value if exists, else -1
  • put(key, value): Insert or update value
  • When cache is full, remove least recently used item
  • Both operations should be O(1)
Solution using OrderedDict: 

from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = OrderedDict()
    
    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        
        # Move to end (most recently used)
        self.cache.move_to_end(key)
        return self.cache[key]
    
    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            # Update existing key
            self.cache[key] = value
            self.cache.move_to_end(key)
        else:
            # Add new key
            if len(self.cache) >= self.capacity:
                # Remove least recently used (first item)
                self.cache.popitem(last=False)
            self.cache[key] = value


Usage

cache = LRUCache(2)
cache.put(1, 1)
cache.put(2, 2)
print(cache.get(1))    # Returns 1
cache.put(3, 3)        # Evicts key 2
print(cache.get(2))    # Returns -1 (not found)
cache.put(4, 4)        # Evicts key 1
print(cache.get(1))    # Returns -1 (not found)
print(cache.get(3))    # Returns 3
print(cache.get(4))    # Returns 4

Solution using Doubly Linked List + HashMap: 

class Node:
    def __init__(self, key=0, value=0):
        self.key = key
        self.value = value
        self.prev = None
        self.next = None

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}  # key -> node
        
        # Dummy head and tail
        self.head = Node()
        self.tail = Node()
        self.head.next = self.tail
        self.tail.prev = self.head
    
    def _add_node(self, node: Node):
        """Add node right after head"""
        node.prev = self.head
        node.next = self.head.next
        self.head.next.prev = node
        self.head.next = node
    
    def _remove_node(self, node: Node):
        """Remove node from list"""
        prev = node.prev
        next = node.next
        prev.next = next
        next.prev = prev
    
    def _move_to_head(self, node: Node):
        """Move node to head (most recently used)"""
        self._remove_node(node)
        self._add_node(node)
    
    def _pop_tail(self) -> Node:
        """Remove tail node (least recently used)"""
        last = self.tail.prev
        self._remove_node(last)
        return last
    
    def get(self, key: int) -> int:
        node = self.cache.get(key)
        if not node:
            return -1
        
        self._move_to_head(node)
        return node.value
    
    def put(self, key: int, value: int) -> None:
        node = self.cache.get(key)
        
        if not node:
            # New node
            new_node = Node(key, value)
            
            if len(self.cache) >= self.capacity:
                # Remove LRU
                tail = self._pop_tail()
                del self.cache[tail.key]
            
            self.cache[key] = new_node
            self._add_node(new_node)
        else:
            # Update existing node
            node.value = value
            self._move_to_head(node)

Question 5: Explain Python's method resolution order (MRO) and multiple inheritance.

Detailed Explanation

Method Resolution Order (MRO):

Python uses C3 linearization algorithm to determine the order in which base classes are searched when looking for a method.

class A:
    def method(self):
        print("A")

class B(A):
    def method(self):
        print("B")
        super().method()

class C(A):
    def method(self):
        print("C")
        super().method()

class D(B, C):
    def method(self):
        print("D")
        super().method()

d = D()
d.method()

Output:


D


B


C


A



MRO: D -> B -> C -> A

print(D.__mro__)

(, , , , )

Diamond Problem: 

class A:
    def method(self):
        print("A")

class B(A):
    pass

class C(A):
    pass

class D(B, C):
    pass

d = D()
d.method()  # Prints "A" - no ambiguity due to MRO
print(D.__mro__)

D -> B -> C -> A -> object

super() Function: super() follows MRO to call the next method in the inheritance chain. 

class Base:
    def __init__(self):
        print("Base init")

class Derived(Base):
    def __init__(self):
        print("Derived init")
        super().__init__()  # Calls Base.__init__

d = Derived()

Output:


Derived init


Base init

Question 6: Explain decorators, generators, and context managers.

Detailed Explanation

Decorators:

Functions that modify or extend the behavior of other functions.

Simple decorator

def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("Before function call")
        result = func(*args, **kwargs)
        print("After function call")
        return result
    return wrapper

@my_decorator
def say_hello(name):
    print(f"Hello, {name}")

say_hello("John")

Output:


Before function call


Hello, John


After function call



Decorator with arguments

def repeat(times):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(times=3)
def greet(name):
    print(f"Hello, {name}")

greet("Alice")

Output: (printed 3 times)


Hello, Alice



Class-based decorator

class CountCalls:
    def __init__(self, func):
        self.func = func
        self.count = 0
    
    def __call__(self, *args, **kwargs):
        self.count += 1
        print(f"Call {self.count} of {self.func.__name__}")
        return self.func(*args, **kwargs)

@CountCalls
def say_hi():
    print("Hi!")

say_hi()  # Call 1 of say_hi
say_hi()  # Call 2 of say_hi

Generators:

Functions that yield values one at a time, memory-efficient.

Generator function

def countdown(n):
    while n > 0:
        yield n
        n -= 1

for num in countdown(5):
    print(num)

Output: 5, 4, 3, 2, 1



Generator expression

squares = (x**2 for x in range(10))
print(list(squares))  # [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]


Memory efficient for large datasets

def read_large_file(filename):
    with open(filename) as f:
        for line in f:
            yield line.strip()


Doesn't load entire file into memory

for line in read_large_file("large_file.txt"):
    process(line)

Context Managers:

Objects that define what happens when entering and exiting a context.

Using with statement

with open("file.txt", "r") as f:
    content = f.read()

File automatically closed



Custom context manager (class-based)

class Timer:
    def __enter__(self):
        self.start = time.time()
        return self
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        self.end = time.time()
        print(f"Elapsed time: {self.end - self.start} seconds")

with Timer():
    time.sleep(1)

Output: Elapsed time: 1.0 seconds



Context manager (function-based)

from contextlib import contextmanager

@contextmanager
def timer():
    start = time.time()
    yield
    end = time.time()
    print(f"Elapsed time: {end - start} seconds")

with timer():
    time.sleep(1)

Detailed Explanation

Async/Await:

Used for concurrent I/O operations without blocking.

Async function

Question 16: Explain Python's collections module.

Detailed Explanation

Collections Module:
  • Specialized container datatypes
  • Extend built-in containers
Key Types:
  • defaultdict: Dictionary with default factory
  • Counter: Count hashable objects
  • deque: Double-ended queue
  • namedtuple: Tuple with named fields
  • OrderedDict: Dictionary that remembers order
Comprehensive Step-by-Step Explanation

Problem Analysis:

This problem requires finding two numbers in an array that sum to a target value. The key challenge is doing this efficiently.

Naive Solution Analysis:

The brute force approach would be to check every possible pair of numbers:

  • For each number at index i, check all numbers at indices j > i
  • If nums[i] + nums[j] == target, return [i, j]
  • Time Complexity: O(n²) - nested loops
  • Space Complexity: O(1) - no extra space

Optimization Strategy:

We can optimize using a hash map (dictionary) to store numbers we've already seen:

  • As we iterate through the array, we store each number and its index
  • For each new number, we calculate what number we need (complement = target - current)
  • We check if this complement exists in our hash map
  • If it does, we've found our pair!

Why This Works:

The key insight is that if we need number X to pair with current number Y to sum to target, and we've already seen X earlier in the array, we can immediately return the answer without checking all remaining numbers.

Example Walkthrough:

For nums = [2, 7, 11, 15], target = 9:

  1. i=0, num=2: complement = 9-2 = 7. Is 7 in map? No. Store {2: 0}
  2. i=1, num=7: complement = 9-7 = 2. Is 2 in map? Yes! Return [0, 1]

Edge Cases to Consider:

  • Empty array: Return empty list
  • No solution: Problem states there's always exactly one, but in real scenarios, handle gracefully
  • Negative numbers: Works fine, complement calculation handles it
  • Duplicate numbers: First occurrence is stored, second occurrence can use it

Example

from collections import Counter, defaultdict

Counter('hello')  # Counter({'l': 2, 'h': 1, 'e': 1, 'o': 1})
dd = defaultdict(list)
dd['key'].append('value')  # No KeyError
